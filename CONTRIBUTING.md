# Contributing to n8n Masterclass

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to this project.

## How Can I Contribute?

### Reporting Bugs

If you find a bug in the workflow or documentation, please open an issue.

### Suggesting Enhancements

We are always looking for ways to make the masterclass better. If you have an idea for a new feature or improvement, feel free to open an issue or submit a pull request.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've modified the workflow, export the JSON and replace the one in `workflows/`.
3. Update the `README.md` if necessary.
4. Ensure your changes follow the project's style and goals.

## Style Guide

- Use descriptive names for nodes.
- Document any complex logic within the node notes if possible.
- Keep the `README.md` updated with any new requirements.

Thanks for your contributions!
